Project Documentation: Minesweeper

Documentation explaining how you implemented your game and how it satisfies the remainder of the requirements. The documentation should be in a file named documentation.html, with a link to the documentation file on the project description page.

The program generates a different mine configuration for each game.
When the player presses generate a grid is created with either the default options or options entered in the input boxes. After the grid is created, a random number of tiles are set as mines.
The player can flag squares in the grid as mines, and can also change a square back to being unflagged.
Toggling the flagging option at the bottom of the page allows a player to flag and unflag tiles
The program tells how many mines are in the grid and displays the number of mines that have been flagged at any given time.
Displays above the grid show how many mines are present and how many flags the player has placed. It does NOT show how many flags are correctly placed, because that would allow cheating.
If a zero square is clicked, all adjacent zero squares (recursively) are also revealed. All non-zero squares adjacent to the revealed zero squares are also revealed.
When a zero tile is revealed, it recursively calls the revealing function on all adjacent tiles. This repeats until all connected zero tiles, and their numbered border tiles, are revealed.
The player can choose the size of the puzzle and the number of mines.
Input boxes are available at the top of the grid that allow the player to customize the game. If too many mines are entered, the game attempts to place as many as it can until it fails (attempts to place a mine on an existing mine) ten times. This means that completely filling up the grid is unlikely, as the mine placing code will fail before it selects every tile. The mine display is accurate to the number of mines actually placed in the grid.
The program displays a message when the game is over (a mine is clicked, or all non-mine squares have been revealed and all mine squares are correctly flagged).
When an end condition is reached (either all tiles are clicked and all mines are flagged, or a mine is clicked) the grid becomes unclickable and a victory/defeat message appears above the grid

Other Comments

I did some work with getting the grid to show up properly with higher dimensions, the div now grows in width if the grid becomes too big. This means that scrollbars appear if it gets too large, but the tiles stay the same size. I feel the need to scroll on oversized grids is an acceptable tradeoff for tile size consistency.
